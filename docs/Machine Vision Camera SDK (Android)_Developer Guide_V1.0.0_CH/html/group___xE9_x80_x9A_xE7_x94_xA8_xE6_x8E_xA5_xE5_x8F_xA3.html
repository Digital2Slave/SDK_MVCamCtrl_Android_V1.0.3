◆ MV_CC_RegisterExceptionCallBack()

public static native int MV_CC_RegisterExceptionCallBack ( Handle  handle,
CameraExceptionCallBack  callBack 
)
static

设置异常回调函数

参数
handle[IN] 设备句柄
callBack[OUT] 异常回调对象
返回
成功，返回MV_OK；失败，返回错误码 。
备注
 • 该接口需要在 MV_CC_OpenDevice() 打开设备之后调用。设备异常断开连接后可以在回调里面获取到异常消息，GigE设备掉线之后需要先调用 MV_CC_CloseDevice() 接口关闭设备，再调用 MV_CC_OpenDevice() 接口重新打开设备。
宏定义 宏定义值 含义
MV_GIGE_EXCEPTION_DEV_DISCONNECT 0x00008001 设备断开连接
 • 接口所在类：MvCameraControl
示例
ReconnectActivity.java.

◆ MV_CC_SetImageNodeNum()

public static native int MV_CC_SetImageNodeNum ( Handle  handle,
int  nNum 
)
static

设置图像节点数

参数
handle[IN] 设备句柄
num[IN] 缓存节点个数
返回
成功，返回MV_OK；失败，返回错误码 。
备注
 • 调用该接口可以设置SDK内部图像缓存节点个数，在调用 MV_CC_StartGrabbing() 开始抓图前调用。
 • 在SDK中默认是1个节点。
 • 接口所在类：MvCameraControl
